## 1. 总体业务闭环（七步生命周期）

```mermaid
flowchart TB
  subgraph Driver["司机 / 自提 H5"]
    A["① 预约<br/>选类型 + 时段"]
    B["② 安全培训<br/>视频 + 答题 · 365 天"]
    C["③ 人车登记"]
    D["④ OCR 资质校验"]
    A2["报到 check-in<br/>取得通行码"]
    F1["⑥ 离场作业勾选"]
    G1["⑦ 司机签退"]
  end

  subgraph Gate["门岗作业台"]
    E["⑤ Check In 安检<br/>清单 + 证据包 + 开入场闸"]
    G2["门岗签退"]
    G3["扫码确认离场<br/>开出场闸"]
  end

  subgraph Admin["管理后台 / EHS"]
    X["入场例外双签批准"]
    H["电子归档<br/>YYYYMMDD_车牌"]

  end

  B --> C --> D
  A --> A2
  D --> A2
  A2 -->|准入通过| E
  A2 -->|准入失败| P["access_pending"]

  X -->|批准| E
  E --> O["onsite 在场"]
  O --> F1 --> G1
  G1 --> G2 --> G3 --> H --> N
```

## 2. 到访状态机（完整）

```mermaid
stateDiagram-v2
  [*] --> appointed: POST /visits 预约

  appointed --> inspecting: check-in 准入通过
  appointed --> access_pending: check-in 准入失败

  access_pending --> inspecting: 补齐后再次 check-in
  access_pending --> exception_requested: 门岗申请入场例外

  exception_requested --> onsite: EHS/管理员批准
  exception_requested --> access_pending: 驳回

  inspecting --> onsite: 安检通过并开闸
  inspecting --> rejected: 安检拒绝

  onsite --> departing: 提交离场检查
  departing --> departing: 司机签 / 门岗签
  departing --> completed: checkout/confirm\n双方已签 + 可选通行码校验

  rejected --> [*]
  completed --> [*]: 写入 archive_key
```
