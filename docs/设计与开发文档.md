## 2. 业务背景与目标

### 2.1 建设目标

落实厂区 **EHS 数字化管理**：对运输车辆与客户自提实现「安全合规、流程闭环、无纸化、可追溯、系统自动校验、效率提升」。

品牌与视觉参考厂区品牌色带（紫 / 品红 / 柠绿 / 翠绿），门户与侧栏使用品牌角标。

### 2.2 业务类型（统一流程）

| 类型 ID | 名称 | 说明 |
|---------|------|------|
| `carrier_inbound` | 运输入场（送货） | 承运送货入厂 |
| `carrier_outbound` | 运输出场（提货） | 承运提货出厂 |
| `self_pickup` | 客户自提 | 须预约 + DN；轻量准入，门岗核身份/授权书 |
| `temporary` | 其他临时车辆 | 培训 + 身份证；事由确认 |

> 历史值 `carrier` 自动归一为 `carrier_inbound`。

### 2.3 七步生命周期（产品对齐）

```mermaid
flowchart LR
  A[①预约/扫码进入] --> B[②EHS安全培训]
  B --> C[③车辆人员登记]
  C --> D[④OCR资质校验]
  D --> E[⑤门岗Check In]
  E --> F[⑥作业完成与离场检查]
  F --> G[⑦双签Check Out]
  G --> H[电子归档与通知]
```

| 步骤 | 产品实现 |
|------|----------|
| ① 预约 | 选业务类型 + 半小时时段（容量校验）→ 生成到访单 `appointed` |
| ② 培训 | 视频 + 答题；有效期 **365 天**；有效则跳过；题库支持中/英/德展示 |
| ③ 登记 | 人车主数据；证件上传（驾驶证/资格证/身份证/危化/DN/授权书等） |
| ④ OCR | 模拟 OCR 提取到期日；缺失/过期 → 硬拦截 |
| ⑤ Check In | 司机报到得通行码 → 门岗安检清单 + 开入场闸 → `onsite` |
| ⑥ 作业/离场检查 | 仓管 DN、EHS 离场检查等勾选 → `departing` + `readyForSign` |
| ⑦ 双签离场 | 司机签退 + 门岗签退 → 扫码确认 → 开出场闸 → `completed` |
| 归档 | `archive_key = YYYYMMDD_车牌`；邮件通知仓管（mock）；台账可检索 |

**明确不做（本轮）**：真实人脸/海康大华 SDK、真实 OCR 云、短信/企微、多租户 SSO。设备层保留 Adapter 预留位。

更细的入场 / 离场 / 状态机图见 [系统流程图.md](./系统流程图.md)。

### 3.1 总体结构

```
┌─────────────────────────────────────────────────────────┐

                            │
┌───────────────────────────▼─────────────────────────────┐
│  Server（Express）                                        │
│  routes → access / visitProfiles / ocr / yardOps / devices │
```

## 5. 到访状态机

### 5.1 状态

| 状态 | 含义 |
|------|------|
| `appointed` | 已预约 |
| `access_pending` | 报到未过准入 / 例外驳回 |
| `inspecting` | 待门岗安检 |
| `exception_requested` | 入场例外待 EHS 双签 |
| `onsite` | 已入场 |
| `departing` | 离场检查中 / 待双签 |
| `completed` | 已闭环离场 |
| `rejected` | 安检拒绝 |

### 5.2 主路径转移

```
appointed ──checkin(ok)──► inspecting ──inspect(pass)──► onsite
    │                           │
    └─checkin(fail)──► access_pending ──exception(gate)──► exception_requested
                              ▲                                    │
                              └──────── reject ────────────────────┤
                                                                   └─approve──► onsite

onsite ──depart(步骤不全)──► departing
onsite ──depart(步骤齐)──► departing(readyForSign)
departing ──sign(driver)+sign(gate)──► both_signed
         ──checkout/confirm──► completed(+ archive_key)
```

### 5.3 入场例外（与离场双签区分）

- 配置 `exception_dual_approve=true` 时：门岗申请 → `exception_requested` → EHS/管理员批准后入场。
- 离场双签：`checkout/sign` + `checkout/confirm`，与入场例外独立。

## 7. 核心规则引擎

### 7.1 准入 `evaluateAccess`（`server/src/services/access.js`）

- **承运 inbound/outbound**：承运商/司机/车辆 active + 培训有效 + 驾驶证/资格证 + 行驶证/保险 + 道路运输证；勾选危化则再校验危化许可 + 电子运单。
- **自提**：轻量；车辆可用；勾选 `safetyBrief` 时强制培训。
- **临时**：培训 + 身份证。

输出：`allowed`、`reasons`、`lights`、`riskScore`、`riskLevel`、`fastLane`。

### 7.2 类型配置 `visitProfiles.js`

每种类型定义：`inspectChecklist`、`departCore`、`departOptional`。  
离场必做 = core + 已选 optional（`hazmat` 仅影响准入证件，不进入离场勾选列表）。

### 7.3 场站编排 `yardOps.js`

- 时段容量 `slot_capacity`（默认 4）
- 通行码 `makePassCode`
- 风险 `computeRisk`
- 停留 SLA `dwell_warn_minutes`（默认 90）
- 归档键 / 仓管邮件 stub

### 7.4 OCR `ocr.js`

支持：`driver_license`、`vehicle_license`、`qualification`、`transport_permit`、`insurance`、`id_card`、`hazmat_permit`、`manifest`、`delivery_note`、`auth_letter`。

## 8. API 契约（摘要）

Base：`/api`。鉴权：Header `Authorization: Bearer <userId>`（演示 token = 用户 id）。

### 8.1 元数据 / 看板

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查 |
| GET | `/meta/visit-types` | 业务类型与清单 |
| GET | `/meta/slots` | 当日时段余量 |
| GET | `/meta/yard-config` | 容量 / SLA / 双签开关 |
| GET | `/dashboard` | 运营 KPI |
| GET | `/gate/kpi` | 门岗 KPI |
| GET | `/notifications` | 司机通知条 |

### 8.2 培训 / 证件 / 准入

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/training/course` | 课程与题目 |
| POST | `/training/progress` | 视频进度 |
| POST | `/training/quiz` | 答题 |
| POST | `/documents/ocr` | 上传识别 |
| POST | `/access/evaluate` | 准入预检 |

### 8.3 到访生命周期

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/visits` | 筛选：`status,type,plate,pickupRef,from,to,archiveKey,passCode` |
| POST | `/visits` | 创建预约（须 `slotStart/slotEnd`） |
| POST | `/visits/:id/checkin` | 报到 |
| POST | `/visits/:id/inspect` | 门岗安检开闸 |
| POST | `/visits/:id/depart` | 提交离场检查 → 待双签 |
| POST | `/visits/:id/checkout/sign` | `{ role: "driver"\|"gate" }` |
| POST | `/visits/:id/checkout/confirm` | 门岗确认离场（可带 `passCode`） |
| POST | `/visits/:id/exception` | 入场例外申请 |
| POST | `/visits/:id/exception/approve\|reject` | EHS 审批 |

### 8.4 设备

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/devices` | 设备列表 |
| POST | `/devices/lpr/simulate` | LPR 模拟；返回 `matchedVisit` |
| POST | `/devices/:id/execute` | 执行开闸等 |

## 9. 前端结构

```
web/src/
  api.js                 # Pages→mock / 本地→HTTP；错误按语言本地化
  mockApi.js             # 演示全量 Mock（STORE_KEY=cgs-pages-demo-v7）
  i18n/
    messages.js          # zh / en / de UI 文案
    I18nContext.jsx      # LangSwitch · BrandLockup
    labels.js            # 状态/类型/证件/风险标签
    content.js           # 培训题库与演示名本地化
  components/
    BrandMark.jsx        # 品牌角标
    StageBands.jsx       # 门户/登录色带背景
    PassCode.jsx
  pages/
    Portal / Login
    driver/              # Home Training Docs Visit
    gate/                # Console Onsite
    admin/               # Dashboard Visits Documents Masters Devices Audit
  styles.css             # Merck 色板 + 响应式
  assets/merck-mark.png  # 角标资源
```

### 9.1 关键交互

- **司机 Visit**：选类型/时段 → 预约 → 报到通行码 → 离场检查 → 司机签退。
- **门岗 Console**：KPI、风险排序、输码/LPR、安检证据包、入场例外。
- **门岗 Onsite**：在场 SLA + 离场双签与确认开闸。
- **台账 Visits**：类型/状态/车牌/DN/归档号检索。

### 9.2 多语言

| 语言 | 代码 | 说明 |
|------|------|------|
| 中文 | `zh` | 默认（非 en/de 浏览器） |
| English | `en` | 全量 UI + 培训题 + 常见错误 |
| Deutsch | `de` | 同上 |

切换控件位于门户、登录、侧栏与司机头栏。品牌主标题随语言切换，副标题为另一语言对照（`brand` / `brandAlt`）。
